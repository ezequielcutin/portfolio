A local-first, no-vendor-lock-in CLI for syncing encrypted secrets across environments. Built for developers who got tired of manually managing .env files across staging/prod and wondering if their secrets were actually encrypted. Because "cloud-based secret manager" shouldn't require a second mortgage and a prayer.

Key Features (Planned)

  • AES-256-GCM encryption with per-secret IVs — authenticated encryption that prevents tampering, stored as portable vault YAML
  • Local-first design: key file generation + vault stored in git (key stays in ~/.secrets), zero friction after init
  • secrets pull/push — sync vault ↔ .env in one command, with conflict detection and interactive prompts
  • Multi-environment support — manage default/staging/prod secrets in a single vault, with secrets diff to catch drift
  • Integration scaffolding — sync to Vercel, GitHub Actions, and beyond without vendor lock-in
  • Audit trail — append-only JSONL log of who changed what, when
  • Zero external dependencies for crypto (Node.js built-in crypto module only)

Current Phase: Project scaffolding (Phase 0). Nailing the encryption design, CLI architecture, and vault format first. Shipping npm package by week 8.

Why this exists: Every team I know has a Notion doc with prod secrets, or uses a free tier SaaS that got hacked, or just... doesn't rotate keys. This is the CLI I'd want to use.
